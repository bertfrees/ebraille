This vocabulary defines properties for describing braille works.

Introduction

About this vocabulary

This vocabulary defines braille-specific properties for describing eBraille publications [[ebraille]].

Referencing

The base URL for referencing this vocabulary is https://www.daisy.org/tbd

For compatibility with the EPUB 3 package document, this specification reserves the prefix "brl:" for use with properties in this vocabulary.

Until such time as the prefix is formally registered as an EPUB 3 reserved prefix, it MUST be declared in the eBraille package document.

Property field definitions

The fields in the vocabulary definition tables have the following implicit requirements:

Allowed Values

Specifies the REQUIRED type of value using [[xmlschema-2]] datatypes.

Cardinality

Specifies the number of times eBraille creators MAY specify the property.

Properties with a minimum cardinality of one MUST be specified.

Description

Describes the purpose of the property and specifies any additional usage requirements that eBraille creators must follow.

Example

Provides non-normative usage examples.

Name

Specifies the name of the property as it MUST appear in the metadata.

Bibliographic properties

completeTranscription

Definition of the completeTranscription property
Name: completeTranscription
Description: Identifies whether the original work being transcribed is fully represented in the braille rendition, minus any minor omissions such as illustrations without captions or other material that is not ordinarily transcribed.
Allowed value(s): true | false
Cardinality: Exactly one
Example: 
<meta property="brl:completeTranscription">true</meta>

producer

Definition of the producer property
Name: producer
Description: Identifies the name of the organization(s) that produced the braille publication.
Allowed value(s): xsd:string
Cardinality: Zero or more
Example: 
<meta property="brl:producer">APH</meta>

volumes

Definition of the volumes property
Name: volumes
Description: Identifies the total number of volumes within the eBraille publication. The number typically corresponds to the number of XHTML content documents in the publication.
Allowed value(s): xsd:unsignedInt
Cardinality: Zero or one
Example: 
<meta property="brl:volumes">3</meta>

Content properties

dotNumber

Definition of the dotNumber property
Name: dotNumber
Description: Identifies whether 6 or 8 dot braille cell characters are used in the text.
Allowed value(s): 6 | 8
Cardinality: Exactly one
Example: 
<meta property="brl:dotNumber">6</meta>

graphicType

Definition of the graphicType property
Name: graphicType
Description:

Identifies the format(s) of any tactile graphics included in the work.

Repeat the property for each different format.
Allowed value(s): JPG | PDF | PNG | SVG
Cardinality: One or more
Example: 
<meta property="brl:graphicType">SVG</meta>

minimumCells

Definition of the minimumCells property
Name: minimumCells
Description:

Identifies the minimum number of cells per line required to accurately display the material within the transcription.

For example, if a single graphic in the file requires at least 10 cells of space and so the minimum for this file would be 10 cells.

Reading systems should attempt to display the content regardless of the value of this property.
Allowed value(s): xsd:unsignedInt
Cardinality: Exactly one
Example: 
<meta property="brl:minimumCells">10</meta>

minimumLines

Definition of the minimumLines property
Name: minimumLines
Description:

Identifies the minimum number of lines per page required to accurately display the material within the transcription.

For single- and multi-line braille displays, the concept of a page refers to the total number of lines that the hardware can accommodate without refreshing.
Allowed value(s): xsd:unsignedInt
Cardinality: Exactly one
Example: 
<meta property="brl:minimumLines">5</meta>

tactileGraphics

Definition of the tactileGraphics property
Name: tactileGraphics
Description: Identifies whether tactile graphics are present in the work.
Allowed value(s): true | false
Cardinality: Exactly one
Example: 
<meta property="brl:tactileGraphics">true</meta>
